# Daily Notes - 2026-04-30

## Fokus Hari Ini

Memulai eksplorasi kerentanan sisi server (Server-Side) dengan fokus pada **SSRF (Server-Side Request Forgery)** menggunakan Python Flask.

## Pelajaran Penting

1. **Server as a Proxy:** Memahami bagaimana server bisa dipaksa menjadi perantara untuk menyerang dirinya sendiri atau infrastruktur internal lainnya.
2. **Trust Boundary:** Menyadari bahwa pengecekan IP (`127.0.0.1`) saja tidak cukup aman untuk melindungi area admin jika aplikasi memiliki celah SSRF.
3. **Exploitation Reach:** Belajar bahwa SSRF bukan sekadar membaca data, tapi bisa digunakan untuk memetakan port (port scanning) di dalam jaringan privat.

## Next Step

Mencoba teknik **Bypass SSRF Filter** (seperti menggunakan DNS Rebinding atau IP Encoding) untuk menghadapi sistem yang memiliki proteksi dasar.
