# Daily Notes - 2026-05-05

## Fokus Hari Ini

Eksploitasi LFI dengan proteksi ekstensi file di lingkungan Windows.

## Progres & Temuan

1. **Analisis OS vs Exploit:** Menemukan bahwa Windows NTFS dan Python 3.x sangat kebal terhadap teknik klasik seperti *Null Byte* (%00) dan *Path Truncation*. Karakter ilegal seperti `?` dan `#` juga langsung ditolak oleh sistem file.
2. **Logic Evasion:** Berhasil melakukan bypass menggunakan kelemahan logika manipulasi string (*slicing*). Dengan memahami cara aplikasi "membersihkan" input, payload dapat disesuaikan agar hasil akhirnya tetap mengeksekusi *directory traversal*.
3. **Hasil:** Berhasil membaca source code `app_secure.py` (Status 200 OK).

## Pelajaran Penting

- Jangan pernah mempercayai input pengguna, bahkan jika sudah difilter dengan ekstensi tertentu.
- Logika aplikasi yang buruk (seperti pemotongan karakter manual) jauh lebih berbahaya daripada ketiadaan filter sama sekali karena memberikan "rasa aman palsu" bagi developer.
