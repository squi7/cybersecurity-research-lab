## Materi yang Dipelajari

### 1. Context Breakout (Menghancurkan Tanda Kutip)

Mempelajari bahwa XSS bukan hanya tentang menyisipkan tag baru, tapi memanipulasi tag yang sudah ada. Jika input berada di dalam atribut:
` <input value="[INPUT]"> `
Karakter tanda kutip (") digunakan untuk menutup atribut tersebut secara paksa.

### 2. Teknik Eksploitasi 1: Event Handler Injection

Menambahkan atribut interaksi seperti `onmouseover` atau `onfocus`.

- Payload: `" onmouseover="alert(1)`
- Hasil: `<input value="" onmouseover="alert(1)">`

### 3. Teknik Eksploitasi 2: Tag Breakout

Menutup tag pembuka sepenuhnya untuk menyisipkan tag baru yang self-executing.

- Payload: `"><img src=x onerror=alert(1)>`
- Hasil: `<input value=""><img src=x onerror=alert(1)>">`

---

## Sink Berbahaya Hari Ini

- **HTML Attribute Context**: Menggabungkan string secara manual untuk atribut adalah celah fatal jika karakter " atau ' tidak di-escape.
- **innerHTML**: Digunakan untuk merender string yang mengandung atribut hasil injeksi.

---

## Cara Fix Aman

### 1. Element Property Assignment

Jangan membangun tag sebagai string. Gunakan properti DOM secara langsung agar browser melakukan encoding secara otomatis.

```javascript
const el = document.createElement('input');
el.value = userInput; // Aman, tanda kutip dianggap teks murni
```

### 2. HTML Entity Encoding

Jika harus menggunakan string, ubah " menjadi `&quot;` sebelum dimasukkan ke dalam template.

---

## Pelajaran Penting

1. **Context is King**: Strategi eksploitasi berubah total tergantung di mana input mendarat (di antara tag vs di dalam atribut).
2. **Interaction Severity**: Bug yang membutuhkan interaksi (onmouseover) biasanya memiliki skor bounty lebih rendah dibanding yang self-executing (onerror).
3. **No Script Needed**: Terbukti bisa mendapatkan XSS tanpa pernah mengetikkan kata "script".
