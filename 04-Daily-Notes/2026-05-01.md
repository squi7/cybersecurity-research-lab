# Daily Notes - 2026-05-01

## Fokus Hari Ini

Mengeksplorasi teknik **Bypass Filter SSRF** pada aplikasi berbasis Python/Flask di lingkungan Windows.

## Progres & Temuan

1. **Identifikasi Masalah:** Menemukan bahwa filter berbasis *Blacklist* sangat rapuh jika hanya mengandalkan pengecekan string statis.
2. **Debugging Challenge:** Menghadapi error `NoneType` (karena parameter kosong) dan `NameResolutionError` pada Windows saat mencoba payload desimal.
3. **The Breakthrough:** Berhasil membypass filter menggunakan alias `http://loopback:5000/`. Teknik ini berhasil karena kata kunci tersebut tidak ada di blacklist namun tetap mengarah ke IP internal.

## Pelajaran Penting

- Mengetahui perbedaan perilaku resolusi IP antara Windows dan Linux. Di Windows, alias seperti `loopback` bisa menjadi celah yang sangat efektif.
- Pentingnya menambahkan pengecekan `if not target_url` untuk mencegah aplikasi crash (Error 500) saat menerima input kosong.
